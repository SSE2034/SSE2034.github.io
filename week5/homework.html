Homework5: Implementing stringlib.hh

In this homework, you will implement several useful string functions that are not included in the C++ string library.

We offer you the documentations and code examples you wiil build. Read and code them on file.

Important Cautions
  • Do not include other libraries.
  • Do not use a function, std::to_string() of <string> library.
Function Name Descriptions

startsWith(string str, char prefix) Returns true if the string str starts with a specific character.

endsWith(string str, char suffix) Returns true if the string str ends with a speicific character.

bool startsWith(string str, string prefix);
bool startsWith(string str, char prefix);
Returns true if the string str starts with the specified prefix, which may be either a string or a character.
std::string str = "Hello World!";
std::string start_str = "Hel";

if(startsWith(str, start_str))
    std::cout << "true" << std::endl;
else
    std::cout << "false" << std::endl;

// Output: "true"

bool endsWith(string str, string suffix);
bool endsWith(string str, char suffix);
Returns true if the string str ends with the specified suffix, which may be either a string or a character.
std::string str = "Hello World!";
std::string end_str = "rld!";

if(endsWith(str, end_str))
    std::cout << "true" << std::endl;
else
    std::cout << "false" << std::endl;

// Output: "true"

string integerToString(int n);
Converts an integer n into the corresponding string. For example, calling integerToString(123) returns the string "123".
int i = 3291850;

std::string str = integerToString(i);

std::cout << str << std::endl;

// Output: "3291850"

string realToString(double d);
Converts a floating-point number into the corresponding string form. For example, calling realToString(23.45) returns the string "23.45".
double d = 3291.1805

std::string str = realToString(d);

std::cout << str << std::endl;

// Output: "3291.1805" or "3291.1084" etc..
Relaxation of implementation

C++에서 한치의 오차없는 소수값을 갖는것은 불가능 합니다. 어느정도의 오차는 발생할 수 밖에 없습니다. 한치의 오차 없이 문자열로 변환하는것은 난이도가 높아 다음과 같이 조건을 완하합니다. 구현에 참고하시길 바랍니다.

  • 채점시 입력은 소수점 4번째 자리가 최대입니다. 그 보다 더 깊은 수는 테스트하지 않습니다.
  • 가장 깊은 소수점 수를 기준으로 -1 ~ +1까지의 오차를 허용합니다. 예를들어 18.05를 변환하는 경우 18.04와 18.06모두 정답으로 인정합니다.
  • 자릿수를 반드시 지킬 필요 없습니다. 예를 들어 18.05를 변환할 경우 반드시 소수점 2자리로 끊지않아도 됩니다. "18.0500"도 정답으로 인정합니다.
